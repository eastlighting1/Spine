# 보안 정책 (Security Policy)

Spine은 아직 초기 단계의 라이브러리입니다. 프로젝트의 뼈대와 주요 주변 구조는 이른바 '바이브 코딩(vibe-coding)' 스타일의 반복적인 시도를 통해 구축되었으며, 이후 테스트, 검증 규칙, 패키징 점검 및 문서 검토를 통해 다듬어졌습니다. 따라서 저희는 보안 피드백, 정확성 관련 의견, 그리고 강화 방안에 대한 제안을 언제나 환영하며, 이를 초기 단계부터 적극적으로 반영하고자 합니다.

## 취약점 보고 방법

Spine에서 보안 문제를 발견했다고 판단되는 경우, 공개된 장소에 상세한 익스플로잇(exploit) 코드, 비밀번호(secrets), 토큰 또는 비공개 인프라 정보를 게시하지 말아 주시기 바랍니다.

본 저장소의 대부분의 보고 사항은 GitHub 기본 협업 기능을 권장합니다:

- **GitHub 이슈(Issue):** 공개적으로 논의 가능한 잠재적 약점, 안전하지 않은 동작, 검증 누락, 패키징 문제 또는 의존성 위험 요소 등을 발견한 경우 이용해 주세요.
- **풀 리퀘스트(Pull Request):** 구체적인 보안 강화 패치, 테스트 케이스 또는 문서 업데이트가 준비된 경우 이용해 주세요.
- **보안 보고 채널(Security Reporting):** 아직 공개적으로 드러내기 민감한 사안이며 해당 채널이 활성화되어 있는 경우 이용해 주세요.

이슈, PR 또는 비공개 보고서를 작성하실 때는 다음 내용을 포함해 주세요:

- 문제에 대한 간략한 설명
- 영향을 받는 버전 또는 커밋 범위
- 재현 단계 또는 개념 증명(PoC)
- 잠재적 영향력

보안 버그인지 단순 정확성 버그인지 확실하지 않더라도, 일단 이슈를 생성한 뒤 '보안 우려 사항(suspected security concern)'임을 명확히 표시해 주시면 감사하겠습니다.

## 지원 버전

보안 수정 사항은 최신 개발 버전을 우선적으로 반영합니다. 명시적인 언급이 없는 한, 이전 버전에는 수정 사항이 백포트(backport)되지 않을 수 있습니다.

## 공개 프로세스

보고가 접수되면 메인테이너는 다음 절차에 따라 진행합니다:

1. 접수 확인 알림
2. 보고 내용 검증
3. 수정 또는 완화 조치 준비
4. 해당되는 경우 릴리스 노트에 변경 사항 기록
